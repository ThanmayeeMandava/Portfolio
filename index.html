Overview

This project investigates and compares the performance of two advanced regression methods‚ÄîRandom Forest Regressor and Gradient Boosting Regressor‚Äîimplemented via Python's scikit-learn and XGBoost libraries. The models are evaluated across three real-world datasets: Indian rainfall, metro traffic volume, and online shopping data. The goal is to understand how library choice and dataset characteristics influence machine learning performance.

Datasets

  • India Monthly Rainfall: Historical data (Kaggle, 1901‚Äì2002)
  • Metro Interstate Traffic Volume: Urban traffic & weather (UCI ML Repository)
  • Online Shopping Dataset: E-commerce transactions (UCI ML Repository, Clothing Retailer 2008)

Note: Datasets must be downloaded separately and placed in a data/ folder in the project root due to licensing restrictions.

Methodology

  • Data Cleaning: Removal of duplicates, missing values, irrelevant features, and outliers
  • Feature Selection: Selection of statistically significant variables
  • Normalization: Applied StandardScaler to numeric features
  • Train/Test Split: 70% training and 30% testing
  • Modeling: Implemented Random Forest and Gradient Boosting with scikit-learn and XGBoost
  • Evaluation Metrics: RMSE, MSE, MAE, R¬≤ Score
  • Comparisons: Performance evaluated per method-dataset pair

Results (Summary)

  • Rainfall Data: Sklearn Gradient Boosting achieved best accuracy (R¬≤ = 0.99)
  • Traffic Volume Data: XGBoost Gradient Boosting slightly outperformed others
  • Online Shopping Data: Sklearn Random Forest delivered highest performance (R¬≤ = 0.98)

Overview

This project provides a containerized environment for MongoDB and PostgreSQL using Docker, enabling seamless setup and management of databases for a drug overdose dataset or application. It simplifies the development and deployment process using docker-compose.

Docker Setup Instructions

  1. Open Command Prompt/Terminal: Use Command Prompt on Windows or Terminal on Mac/Linux.
  2. Navigate to Project Directory: Go to the folder containing the docker-compose.yml file.
  3. Build and Run Docker Containers: 
    docker-compose -p group_o up --build -d

    This builds the necessary images and starts the containers in detached mode.

  4. Verify Container Status: 
    docker ps

    Use this to ensure both MongoDB and PostgreSQL containers are running.

  5. Database Configuration: Once the containers are live, update your app‚Äôs config to connect to the running databases.

PostgreSQL Instruction

  • Create a new database within the PostgreSQL container.
  • Use pgAdmin or any preferred PostgreSQL client to connect and begin working with your schema or dataset.
